## 定义

Obsidian 笔记库在多设备间的同步方案，通过 Git 版本控制系统实现文本内容的可靠同步、冲突检测和历史版本追踪。

## 关联来源

[[Obsidian 的 PC 端同步方案，无代码搞定 Git 同步]]

## 已知边界 / 局限

- **优势**：Git 天然适合文本类内容的版本管理，支持冲突检测、历史回溯，定时同步减少冲突；多设备间同步无额外费用
- **不适用场景**：超大仓库（GB 级 PDF、媒体文件），受 GitHub 单文件 100MB / Gitee 50MB 限制；非文本内容（图片、PDF）可同步但不保留变更历史
- **冲突处理**：定时同步（非实时）可减少冲突，但无法完全消除；冲突需手动解决

## 实际案例

- **Obsidian Git 插件**：内置提交/拉取/推送操作，适合非技术用户
- **GitHub Desktop**：图形化 Git 操作，降低使用门槛
- **Gogs.io 本地部署**：可在 NAS 上运行，资源消耗低

## 版本 / 演进

- 方案成熟：Git 同步已成为 Obsidian 社区最推荐的跨平台同步方案之一
- 与 iCloud Drive 的对比：建议不要同时使用 iCloud 和 Git，避免双重同步导致冲突
- 手机端方案：iOS 可通过 Git + Shortcuts 实现自动同步
