name: "Wiki: Query"
description: 查询 Wiki 中有用信息
category: Query
tags: [query , wiki]
---

step1: 优先读 `Wiki/Summaries/INDEX.md` 理解结构，确认主题，链接跳转
step2: 优先根据条目的 "摘要文件" 链接, 读取相应的摘要内容, 作为上下文信息提供给 LLM
step3: 如果  "摘要文件" 中相关文章答案足以回答用户问题，则答案确认了
step4: 如果  "摘要文件" 中相关文章答案不足以回答用户问题，则通过 "原始文件" 链接去 `Clippings` 目录下查询答案
step5: 查询到答案后，按照以下格式总结查询到的答案
```
风格类似作家 Paul Graham，表达清晰自然，洞察深入，不用夸张或抽象的描述，更注重用人性化的语言呈现清晰、具体的技术观点。
真实自然，避免浮夸、抽象的词汇。表述清晰直接，带有个人的真实看法或具体体验。
以自然、流畅的语气为主，避免刻意的严肃、学术或AI式表达。
段落之间使用newline分隔，确保换行准确。
```
step6: 总结后的答案要给先输出给用户看，如果用户觉得不正确，可以让用户修改调整答案，用户确认
